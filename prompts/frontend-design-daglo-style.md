# 다글로(Daglo) 스타일 프론트엔드 디자인 프롬프트

## 설명
daglo.ai의 모던하고 기술 중심적인 디자인 언어를 레퍼런스하여 프로덕션 급의 UI/UX를 구현합니다.
표절하지 않으면서도 비슷한 감성을 유지하는 것이 목표입니다.

## 핵심 디자인 원칙

### 1. 레이아웃 구조
- **싱글 페이지 리얀드 페이지 아키텍처**: 스크롤 기반의 섹션별 콘텐츠 노출
- **세로 흐름 우선**: 모바일-우선 반응형 디자인
- **명확한 섹션 구분**: 여백과 색상 변화로 시각적 계층 구성
- **고정 네비게이션**: 상단 네비게이션 바는 스크롤 시에도 접근 가능

### 2. 컬러 팔레트
- **다크모드/라이트모드 지원**: CSS 변수로 테마 전환 지원
- **뉘앙스 있는 중립색**: 순수한 검은색/흰색보다는 자연스러운 톤
- **강조색 사용 최소화**: 중요한 CTA와 강조가 필요한 부분에만 사용
- **대비율 확보**: 접근성 WCAG 2.1 AA 기준 이상 충족

### 3. 타이포그래피
- **계층적 텍스트 구조**: H1 > H2 > H3 > Body > Small 명확한 구분
- **읽기 편한 라인 높이**: 1.6em 이상
- **적절한 글자 크기**: 데스크톱 16px, 모바일 14px 이상
- **산세리프 폰트 선호**: 모던하고 깔끔한 느낌

### 4. UI 컴포넌트
- **카드 기반 레이아웃**: 기능, 사용 후기, 기능 설명을 카드 형식으로 구성
- **부드러운 그림자**: 깊이감 있지만 과하지 않은 shadow
- **라운드 코너**: 8px~12px 정도의 moderate border-radius
- **마이크로 인터랙션**: hover, focus 상태에 미묘한 애니메이션
- **명확한 CTA 버튼**: 강조색을 사용한 버튼, hover 상태에서 스케일/색상 변화

### 5. 접근성과 UX
- **키보드 네비게이션**: 모든 인터랙티브 요소가 탭으로 접근 가능
- **스크린 리더 지원**: 의미 있는 alt 텍스트, aria 레이블 사용
- **포커스 표시**: 명확한 포커스 아웃라인
- **모션 선호도 존중**: prefers-reduced-motion 지원

### 6. 성능과 최적화
- 이미지 최적화: WebP 포맷, responsive images
- 지연 로딩: 스크롤 시 이미지/콘텐츠 로드
- 애니메이션 최적화: GPU 가속, 60fps 유지

## 구현 가이드

### 색상 설정 예시
```css
:root {
  /* Light mode */
  --bg-primary: #ffffff;
  --bg-secondary: #f9f9f9;
  --text-primary: #1a1a1a;
  --text-secondary: #666666;
  --accent-color: #5B5FFF; /* 또는 프로젝트의 강조색 */
  --border-color: #e5e5e5;
}

@media (prefers-color-scheme: dark) {
  :root {
    --bg-primary: #1a1a1a;
    --bg-secondary: #2a2a2a;
    --text-primary: #ffffff;
    --text-secondary: #aaaaaa;
    --border-color: #404040;
  }
}
```

### 컴포넌트 작성 원칙
- React/Next.js 기반 컴포넌트 개발
- Tailwind CSS 또는 CSS-in-JS 활용
- 재사용 가능한 작은 컴포넌트 조합
- TypeScript로 타입 안전성 확보
