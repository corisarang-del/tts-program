1. 제품 개요 (Executive Summary)
1.1 제품명
상황 기반 즉시 소통 서비스 (가칭: QuickTalk)
1.2 제품 목적
사용자가 '상황'과 '의도'만 선택하면 즉시 사용할 수 있는 문장과 음성(TTS)을 제공하여, 말로 설명하기 어렵거나 문장 작성 시간이 부족한 상황에서 효과적인 소통을 지원합니다.
1.3 핵심 가치 제안
• 최소 클릭: 2~3번의 선택만으로 완성된 문장 제공
• 즉시성: 선택 즉시 결과 제공, 고민 시간 제로
• 접근성: 음성 출력(TTS)으로 다양한 사용 환경 지원
• 학습 시스템: 사용자 행동 로그 기반 서비스 개선
1.4 핵심 설계 원칙
• 선택지는 최대 3~4개로 제한
• 클릭 수 최소화
• 결과는 즉시 제공 (로딩 1초 이내)
• 모든 사용자 행동은 로그로 기록
• 개인정보 최소 수집 원칙 준수
2. 타깃 사용자 (Target Users)
2.1 주요 페르소나
페르소나 1: 바쁜 직장인 (김현수, 32세)
• 특성: 회의, 미팅 등 빠른 응답이 필요한 상황이 많음
• Pain Point: 격식 있는 문장을 빠르게 작성하기 어려움
• 사용 시나리오: 회의 지각, 일정 변경 요청, 긴급 보고 등
페르소나 2: 사회 초년생 (이지은, 25세)
• 특성: 직장 예절 문장 작성 경험 부족
• Pain Point: 어떻게 말해야 할지 고민하는 시간이 김
• 사용 시나리오: 상사에게 요청, 동료와의 협업 메시지 등
페르소나 3: 외국어 학습자 (Alex, 28세)
• 특성: 한국 직장 생활 중 한국어 표현이 서툼
• Pain Point: 문화적으로 적절한 표현 찾기 어려움
• 사용 시나리오: 일상적인 직장 커뮤니케이션 전반
2.2 사용자 요구사항
• 반복적으로 비슷한 상황을 겪는 사용자
• 말로 설명하기 어렵거나 문장 고민 시간이 없는 사용자
• 상황에 맞는 적절한 표현을 찾고 싶은 사용자
• 음성으로 전달해야 하는 환경의 사용자
3. 핵심 사용자 흐름 (User Flow)
3.1 전체 플로우 (고정)
랜딩 → 상황 선택 → 의도 선택 → 문장 생성 → TTS 재생 → 결과 평가 → 재사용
3.2 단계별 상세 설명
Step 1: 랜딩 (0초)
• 서비스 소개 및 사용 방법 안내
• '시작하기' 버튼 클릭
• 로그 수집 동의 안내 (선택적)
Step 2: 상황 선택 (3초 이내)
• 3~4개 사전 정의된 상황 카드 표시
• 클릭 → 상태 저장 → 자동 이동
• 예시: '회의 지각', '일정 변경', '도움 요청', '보고 사항'
Step 3: 의도 선택 (3초 이내)
• 선택한 상황에 맞는 3~4개 의도 버튼 표시
• 클릭 → API 호출 → 문장 생성
• 예시 (회의 지각 상황): '사과', '도착 시간 알림', '온라인 참여 요청'
Step 4: 문장 출력 (1초 이내)
• 생성된 문장 세트 표시 (2~3개 옵션)
• TTS 재생 버튼
• 복사 버튼
• '다시 생성' 옵션
Step 5: TTS 재생 (즉시)
• 선택한 문장을 음성으로 출력
• 재생/일시정지 컨트롤 제공
Step 6: 결과 평가 (선택적)
• '해결됨 / 보통 / 도움 안됨' 선택
• 피드백 로그 저장
• 건너뛰기 가능
Step 7: 재사용 (선택적)
• '다른 상황 선택' 버튼
• '처음으로' 버튼
4. 기능 요구사항 (Functional Requirements)
4.1 페이지별 요구사항
4.1.1 랜딩 페이지
요구사항	상세 설명	우선순위
서비스 소개	한 줄 설명 + 핵심 가치 3가지	P0
사용 방법 요약	3단계 플로우 시각화	P0
시작 버튼	CTA 버튼, 상황 선택으로 이동	P0
로그 수집 안내	개인정보 처리 방침 링크	P1

4.1.2 상황 선택 페이지
요구사항	상세 설명	우선순위
상황 리스트	3~4개 카드형 버튼 (아이콘 + 텍스트)	P0
선택 피드백	클릭 시 시각적 피드백 (강조 효과)	P0
자동 이동	선택 즉시 의도 선택 페이지로 전환	P0
뒤로가기	이전 페이지로 이동 버튼	P1

4.1.3 의도 선택 페이지
요구사항	상세 설명	우선순위
의도 리스트	상황별 3~4개 옵션	P0
API 호출	선택 시 문장 생성 API 호출	P0
로딩 표시	API 응답 대기 중 스피너 표시	P0
상황 컨텍스트	선택한 상황 상단에 표시	P1

4.1.4 문장 출력 페이지
요구사항	상세 설명	우선순위
문장 세트	2~3개 옵션 카드형 표시	P0
TTS 버튼	각 문장마다 재생 버튼	P0
복사 버튼	클립보드 복사 + 토스트 알림	P0
다시 생성	동일 상황/의도로 재생성	P1
공유 기능	카카오톡, 문자 공유	P2

4.1.5 결과 평가 페이지
요구사항	상세 설명	우선순위
평가 옵션	해결됨 / 보통 / 도움 안됨 (3단계)	P0
로그 저장	선택 시 자동 저장	P0
건너뛰기	평가 없이 다음 단계 진행 가능	P0
추가 피드백	'도움 안됨' 선택 시 이유 선택	P2

4.1.6 분석 페이지 (Phase 2)
요구사항	상세 설명	우선순위
사용 요약	총 사용 횟수, 자주 쓴 상황	P2
재사용 유도	즐겨찾기, 빠른 실행	P2
히스토리	최근 생성 문장 목록	P2

4.2 데이터 모델
4.2.1 상황 (Situation)
• situation_id: 고유 ID
• name: 상황명 (예: '회의 지각')
• description: 설명
• icon: 아이콘 리소스
• display_order: 표시 순서
4.2.2 의도 (Intent)
• intent_id: 고유 ID
• situation_id: 연결된 상황 ID
• name: 의도명 (예: '사과')
• description: 설명
• template: 문장 생성 템플릿
4.2.3 사용 로그 (Usage Log)
• log_id: 고유 ID
• user_id: 사용자 ID (익명 가능)
• situation_id: 선택한 상황
• intent_id: 선택한 의도
• generated_text: 생성된 문장
• tts_played: TTS 재생 여부
• result_rating: 평가 (1-3)
• timestamp: 생성 시간

10. 부록 (Appendix)
10.1 초기 상황/의도 예시
상황 1: 회의 지각
• 의도 1: 사과 → '죄송합니다, 교통 상황으로 늦었습니다. 5분 내로 도착하겠습니다.'
• 의도 2: 온라인 참여 → '현재 이동 중이라 온라인으로 먼저 참여하겠습니다.'
• 의도 3: 일정 조율 → '늦어질 것 같은데, 회의 시간을 조정할 수 있을까요?'
상황 2: 일정 변경 요청
• 의도 1: 공손한 요청 → '갑작스러운 일정이 생겨 다른 시간으로 변경 가능할까요?'
• 의도 2: 대안 제시 → '오늘 오후 3시나 내일 오전 중 가능하실까요?'
• 의도 3: 긴급 변경 → '죄송하지만 긴급 상황으로 오늘은 어려울 것 같습니다.'
상황 3: 도움 요청
• 의도 1: 공손한 부탁 → '시간 괜찮으시면 ~에 대해 도움 부탁드려도 될까요?'
• 의도 2: 긴급 도움 → '급한 상황인데 잠시 도와주실 수 있으신가요?'
• 의도 3: 정보 요청 → '~에 대해 잘 아시는 것 같아 조언 구하고 싶습니다.'
10.2 용어 정의
• TTS (Text-to-Speech): 텍스트를 음성으로 변환하는 기술
• CCU (Concurrent Users): 동시 접속 사용자 수
• MAU (Monthly Active Users): 월간 활성 사용자 수
• NPS (Net Promoter Score): 고객 추천 지수
• P0/P1/P2: 우선순위 레벨 (P0 = 필수, P1 = 중요, P2 = 선택)
